# 🧊 SmartFridge. — Умный Облачный Холодильник с ИИ-Шефом

«SmartFridge» — это современное Fullstack-приложение (SPA) в стиле **Bento UI**, которое помогает пользователям вести учет продуктов в доме, контролировать их сроки годности и автоматически генерировать уникальные кулинарные рецепты с помощью искусственного интеллекта на основе текущего содержимого холодильника.

---

## 🚀 Основные возможности (Features)

- **🔐 Умная авторизация и безопасность (Supabase Auth):** Полноценная клиентская валидация форм на русском языке, защита от повторных кликов, скрытие/показ пароля и мгновенное реактивное переключение экранов. Данные пользователей защищены на бэкенде с помощью политик **RLS (Row Level Security)**.
- **🧊 Инвентарь в стиле Bento UI:** Адаптивная CSS-Grid сетка продуктов. Умный шаг изменения количества в зависимости от единиц измерения (целые числа для _шт/уп_ и десятые доли для _кг/л_), ручной ввод с клавиатуры и автоматический сброс фокуса по клавише `Enter`.
- **🛡 Автоматический контроль просрочки:** Продукты с истекшим сроком годности визуально блокируются в интерфейсе (замораживаются инпуты управления) и автоматически отфильтровываются перед отправкой запроса нейросети.
- **🪄 Интеграция с Генеративным ИИ (OpenRouter API):** Генерация структурированных JSON-ответов от нейросети на основе реального содержимого холодильника.
- **💾 Умное кэширование (Performance Optimization):** Использование паттерна _Lazy State Initialization_ для работы с кэшем в `localStorage`. Сгенерированный рецепт сохраняется при переключении между вкладками и перезагрузке страницы, экономя лимиты API.
- **🛒 Интерактивный список покупок:** ИИ автоматически анализирует рецепт и выводит недостающие ингредиенты. Пользователь в один клик может закинуть их в список покупок, где реализован удобный интерактивный чек-лист.
- **📱 100% Мобильная адаптивность:** Приложение оптимизировано под любые экраны. На смартфонах десктопный Sidebar плавно скрывается, уступая место нативному нижнему таб-бару (Bottom Navigation) в стиле PWA.

## 🛠 Технологический стек (Tech Stack)

### Frontend:

- **React 19** (Functional Components, Hooks)
- **TypeScript** (Строгая типизация интерфейсов, пропсов и событий)
- **Vite** (Супербыстрая сборка и горячая перезагрузка)
- **Tailwind CSS v4** (Современная декларативная верстка и Bento-дизайн)
- **Lucide React** (Пакет минималистичных векторных иконок)

### Backend & Infrastructure:

- **Supabase (BaaS)** — Облачная база данных **PostgreSQL**
- **Supabase Auth** — Сессии, JWT-токены и управление пользователями
- **OpenRouter API** — Интеграция с искусственным интеллектом (универсальный роутер бесплатных моделей)

## 📐 Архитектура проекта (Architecture)

Проект спроектирован по принципу разделения ответственности (**Single Responsibility**). Компоненты разделены на презентационные («глупые») и контейнеры («умные»):

```text
src/
├── components/          # Реализуемые UI-компоненты
│   ├── AiChef/          # Блок искусственного интеллекта (Контейнер, Скелетон, Отображение)
│   ├── Fridge/          # Инвентарь холодильника (Сетка карточек, Bento-карточка)
│   ├── Sidebar/         # Левая Bento-панель и блок профиля
│   ├── ShoppingList/    # Интерактивный чек-лист покупок
│   └── Auth.tsx         # Экран контролируемой авторизации с валидацией
├── services/            # Изолированный бэкенд-слой (Запросы к таблицам Supabase)
├── utils/               # Хелперы (aiService, маппинг категорий еды, клиент базы данных)
├── App.tsx              # Главный рут, управление глобальной сессией
└── main.tsx             # Точка монтирования приложения и TS-декларации
```

## 📦 Как запустить проект локально (Installation)

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com
   cd food-app
   ```

2. Установите все зависимости:

   ```bash
   npm install
   ```

3. Создайте файл **`.env.local`** в корне проекта и добавьте ваши ключи от Supabase и OpenRouter:

   ```env
   VITE_SUPABASE_URL=https://supabase.co
   VITE_SUPABASE_ANON_KEY=your_supabase_anon_key_here
   VITE_OPENROUTER_API_KEY=your_openrouter_api_key_here
   ```

4. Запустите сервер для разработки:
   ```bash
   npm run dev
   ```
   Откройте в браузере адрес: `http://localhost:5173`

## 📜 База данных (Бэкенд-структура)

Таблица `products` в PostgreSQL управляется через Supabase и имеет следующую схему:

- `id` (int8 / uuid) — Первичный ключ.
- `user_id` (uuid) — Внешний ключ на `auth.users` (автоподстановка `auth.uid()`).
- `name` (text) — Название продукта.
- `quantity` (float8) — Количество (поддержка дробных значений).
- `unit` (text) — Ед. измерения (шт, кг, л, уп).
- `days_left` (int8) — Срок годности.
- `category` (text) — ID категории для автоматического маппинга Bento-иконок.
